11.2.5.7 批量导入网格地址

1)     网格地址的批量导入，需严格根据模版填写数据，建议在系统初始化情况下使用，导入大批量数据可能会影响性能。模版格式如下表：

2)     导入的文档格式必须是office 2003的Excel文档。

3)     Excel每个工作表能显示的行数有限，建议分多个工作表，但每一个工作表的第一行必须是网格地址层级名称及地址的管理医生。

4)     模版中按8层完整地址排列，市层级固定故不体现，列名可以更改，每列之间有从属关系因此字段顺序不允许调整，数据可以为空但不能缺少字段，即10列是固定的，不允增加或删除。

5)     文档中数据按顺序排列，即同一个单元的数据放一起，同一幢的放在一起，依次类推，可提高运行速度，且尽量少的避免小区以下属性重复存储。

6)     如果单元格中，全部是数字，则需要在数据前加单引号（'）将其转化成文本格式。

7)     责任医生、儿保医生、妇保医生的内容为相关用户的登录名，可以为空。

8)     小区层级以下内容须填写全整，例如上图模版，不然在建档过程中容易出现101，102类似地址，通过拼音码调取的时候就无法精确定位。
